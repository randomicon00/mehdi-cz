Automate first drafts and keep them delightful.

## Pipeline sketch

```yaml
inputs: commits, PR labels, screenshots, metrics
outputs: changelog.md + product_notes.md
reviewers: eng lead, PM, support
```

## Why this matters

- Clear notes reduce support load and drive adoption.

## How to use this today

- Label PRs by type; generate draft notes in CI.
- Include upgrade steps and known issues sections.

## Common pitfalls

- Internal jargon; keep user language and screenshots.

## What to try next

- A/B test email subject lines.
- Generate localized notes automatically.
